## Implementation Plan

### Step 1: Create Basic HTML Structure
- Copy structural elements from index.html (DOCTYPE, html, head, body)
- Modify navbar colors to match requirements
- Add page heading
- Create section containers for clothing and books

### Step 2: Add CSS Styling
- Define CSS variables for dark green (#1a4a2e) and gold (#c9a84c)
- Style navbar with specified colors
- Style page heading and section headings
- Style product cards with hover effects
- Style price text in gold
- Style order buttons with dark green background and gold text
- Style out-of-stock indicators
- Add responsive design for mobile
- Style loading and error messages

### Step 3: Add Supabase Integration
- Include Supabase CDN script: https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2
- Initialize Supabase client with provided URL and anon key
- Create async function to fetch products from 'products' table

### Step 4: Implement Product Display Logic
- Fetch products from Supabase
- Separate products by category (clothing vs books)
- Create product card elements dynamically
- For each product:
  - Create card container
  - Add product image placeholder (using first chars of name)
  - Add product name (bold)
  - Add description
  - Add price in KSh format with gold color
  - Add Order Now button (links to whatsapp_message) if in_stock is true
  - Add "Out of Stock" message if in_stock is false
- Append cards to appropriate section containers

### Step 5: Implement Loading and Error States
- Show loading message when fetch starts
- Hide loading message when fetch completes (success or error)
- Show error message if fetch fails
- Clear containers before populating with new data

### Step 6: Add Mobile Navigation
- Implement hamburger menu for mobile views
- Toggle navigation links visibility
- Close menu when links are clicked

### Step 7: Initialize Application
- Call fetchProducts function when DOM content is loaded

## File Structure

## Dependencies
- Supabase JS library via CDN
- Google Fonts (Inter and Playfair Display) - same as index.html

## Implementation Notes
1. Use the same CSS patterns from index.html where applicable
2. Maintain consistent spacing and styling with the existing site
3. Handle edge cases:
   - Empty product lists
   - Missing fields (description, price, whatsapp_message)
   - Network errors
4. Use modern JavaScript features (async/await, arrow functions)
5. Ensure accessibility with proper semantic HTML
6. Use placeholder text for product images until actual images are available in database
